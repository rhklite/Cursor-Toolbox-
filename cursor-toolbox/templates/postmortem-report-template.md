# Postmortem Report Template

> **Placeholder variables:** `{report_root}` = report folder path under docs/reports (example: `docs/reports/stability-curriculum-push-ramp-v2_20260325_1500`), `{model}` = model label (example: `model_7027`), `{mode}` = `linear` or `angular`.

---

## Output Contract

> Produce canonical outputs in reports plus an experiments backlink.

- Canonical human report path: {report_root}/postmortem.md
- Canonical LLM bundle path: {report_root}/llm_postmortem_digest/
- LLM bundle includes: DIGEST files, COMPARISON.md when multi-run, diagnosis.md text-only
- Visual artifacts path: {report_root}/graphics/{model}/
- Experiments backlink: docs/experiments/postmortems/YYYYMMDD_HH/postmortem.md -> ../../../reports/{slug}_{ts}/postmortem.md
- All image paths in postmortem.md use repo-root-absolute paths (leading /)

All chart PNGs are generated by postmortem_charts.py via report_assembler.py. Do NOT generate charts ad-hoc.

## Primary Metrics Analysis (Tier 1)

> Interleave findings with visuals for direct evidence mapping.

**Dashboard**

![tier1_dashboard](/{report_root}/graphics/{model}/tier1_dashboard.png)

- Summarize what worked in one to three bullets from Tier 1 evidence.
- Summarize what failed in one to three bullets from Tier 1 evidence.

**Graceful Degradation Rule**

- If dashboard is unavailable, state gate result and reason, then continue with table-only analysis.

**Survival Heatmaps**

For each run label and sweep mode, include the corresponding heatmap:

![survival_heatmap_{model}_{mode}](/{report_root}/graphics/{model}/survival_heatmap_{model}_{mode}_{mode}.png)

- Compare strongest and weakest conditions with one-line implications.
- Repeat for each run label and mode (linear/angular) combination.

**Graceful Degradation Rule**

- If survival heatmaps are unavailable, cite the metric table rows used instead.

## Behavioral Evidence

> Use visual evidence to validate or falsify expected behavior.

**Keyframe or Equivalent Evidence**

For each run, include the keyframe grid from postmortem_digest.py:

![grid_{run_basename}](/{report_root}/graphics/{model}/grid_{run_basename}.png)

- Observed behavior progression for each representative run in one sentence each.
- Behavior versus expectation gap in one to three bullets.

**Graceful Degradation Rule**

- If no keyframes exist, state no behavioral visuals available and continue using digest behavior metrics only.
